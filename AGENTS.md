# AGENTS.md — гид для внесения изменений в `pravo-grammatika`

Прочитайте перед правкой `SKILL.md`, референсов, манифестов, доков или тестов. Файл описывает,
как устроен репозиторий и какие правила нельзя нарушать при контрибьюте.

Это скилл и плагин Claude Code: он проверяет **форму выражения** юридического текста на материале
практики РФ (синтаксическая неоднозначность, пунктуация, область действия, отсылки, модальность,
нормы стиля) и **не проверяет ни валидность вывода** (это `pravo-logika`), **ни убедительность
подачи** (это `pravo-ritorika`). Третий по счёту скилл тривиума грамматика · логика · риторика.

> Терминология (рус.): прилагательное для «agent / agentic» — **«агентный»**, не «агентский».

## Один слой

Единый оригинальный грамматический слой — `references/*.md` + `SKILL.md`. Заимствованного кода или
движка нет; понятийная основа (грамматика, синтаксис, семантика области действия, деонтические
операторы) — общее достояние, см. [`THIRD_PARTY_NOTICES.md`](THIRD_PARTY_NOTICES.md). Деонтический
аппарат (операторы O/P/F) общий с парным `pravo-logika` — но здесь он приложен к **лексико-
грамматической реализации** модальности (какое русское слово какой оператор выражает), а формальные
выводы из операторов остаются за `pravo-logika`.

Порядок разбора зашит в `SKILL.md`: жанр фрагмента → построение разбора и точка ветвления → оба
прочтения через факты и расхождение правового результата → классификация по каталогу → развод
конструкции → фиксация чужой юрисдикции флагами → выбор прочтения оставлен автору.

## Инварианты (не нарушать при правках)

1. **Не проверяет право по существу.** Скилл не подставляет номера статей, механику исчисления
   сроков и позиции судов по памяти — ставит флаг `[требует проверки: норма / практика]`. Любой
   пример или инструкция, заставляющие модель утверждать содержание нормы без источника, — ошибка.
2. **Смысл не додумывает.** При двух прочтениях показываются **оба**, выбор оставляется автору.
   Молча закрепить одно прочтение (в том числе в режиме `rewrite`) — запрещено: это подмена воли
   сторон. В `rewrite` при неустранимой неясности выдаются **два** варианта под выбор, а не один
   угаданный. Любой пример, где скилл «исправляет» неоднозначность выбором за автора, — ошибка.
3. **Форма, а не валидность и не убедительность.** Понятийный дефект (плывущий термин, объём
   понятия, подмена тезиса) фиксируется одной строкой с флагом `[→ pravo-logika]`; вопрос подачи
   (структура, порядок, адресат) — флагом `[→ pravo-ritorika]`. Разбирать их здесь глубже флага —
   ошибка. Граница с логикой — «синтаксис против понятий» (тест двух деревьев, `SKILL.md`).
4. **Каждое замечание — через правовой результат.** Критерий любого дефекта — **как меняется
   объём прав и обязанностей**, а не «правило требует запятую». Замечание без показанного
   последствия для однозначности или правового результата — вкусовое, ему место в отдельном блоке
   низкого приоритета с пометкой «не влияет на смысл» либо вне скилла (граница компетенции 4).
5. **Без персональных данных.** В контенте (примеры, референсы, README, тесты) — никаких реальных
   дел, клиентов, имён, внутренних материалов. Только условные и учебные примеры.
5а. **Терминология — российского права.** Англо-американские правовые институты в контенте не
   используются как термины — берутся российские конструкции: заверения об обстоятельствах,
   возмещение имущественных потерь, договор купли-продажи акций/долей, существенное неблагоприятное
   изменение, сделка с заинтересованностью, крупная сделка, условия закрытия. Допустимы: обозначение
   сферы «M&A», латынь при разборе (ejusdem generis, a fortiori) с русским глоссом, англоязычные
   лингвистические термины с русским объяснением (scope — область действия).
6. **Лицензия и атрибуция.** Проект под **MIT**, автор и правообладатель — Sofya Smirnova
   ([t.me/forgednotwritten](https://t.me/forgednotwritten)); поле `license` в трёх манифестах — `"MIT"`.
   Не ломать `LICENSE` и `THIRD_PARTY_NOTICES.md`. Рядом с
   именем автора везде — ссылка на телеграм-канал. Единственный сторонний ассет — свободный шрифт
   Jun, встроенный base64 в лендинг `docs/index.html` (заголовки, вордмарк); он MIT-лицензией
   проекта не покрывается — оговорка в `THIRD_PARTY_NOTICES.md` §3. При правках лендинга шрифт Jun
   для кириллицы обязателен (иначе падает в системный serif) — не удалять `@font-face`.

## Как править

- **Референс:** назначение (одна строка) + ссылки на смежные файлы + разделы + чек-лист в конце.
  Стиль — сухой, российский правовой язык, без нейросетевых клише. Каждый узел — с правовым
  примером и показом расхождения результата. Ориентир по тону и плотности — парные `pravo-logika` /
  `pravo-ritorika`.
- **Новый дефект в каталоге** (`references/grammar-errors.md`): по шаблону файла — имя дефекта →
  как выглядит → как меняется правовой результат → как развести; с указанием тематического референса.
- **Имя и версия.** Имя скилла — `pravo-grammatika` (поле `name` в `SKILL.md`). При содержательных
  изменениях бампайте `version` синхронно в `SKILL.md` и трёх манифестах (`.claude-plugin`,
  `.codex-plugin`, `.cursor-plugin`).
- **Структура плагина.** Внутри `.claude-plugin/` — только `plugin.json`. Маркетплейс семейства
  вынесен в отдельный репозиторий [`sky-magenta/damascus-ink-plugins`](https://github.com/sky-magenta/damascus-ink-plugins)
  (перечисляет все три скилла через github-источники) — здесь `marketplace.json` нет.
  `commands/`, `references/`, `SKILL.md` — в корне репозитория, не внутри `.claude-plugin/`.
- **Граница с `pravo-logika`.** Если правка размывает границу «синтаксис / понятия» — это ошибка.
  Всё про «следует ли вывод» и «что значит термин по объёму» — чужая юрисдикция, флаг `[→ pravo-logika]`.

## Тесты

Набор E2E — в `evals/` (источник истины — `build_cases.py`).

- После правки примеров: `python evals/build_cases.py` — перегенерирует `cases.jsonl`.
- Обязательно, офлайн: `python evals/validate_cases.py` — проверка структуры и покрытия (должно
  завершиться «OK»); годится и для CI (`pytest evals/validate_cases.py`). На Windows-консоли с
  кодировкой cp1251 запускайте с `PYTHONIOENCODING=utf-8`.
- Полный прогон через модель: `python evals/run_evals.py` (нужен `ANTHROPIC_API_KEY`; `--dry-run`
  собирает промпты без сети).
- Новый тип дефекта — добавляйте примеры: хотя бы один `defect` и один `clean`, чтобы ловить и
  пропуск дефекта, и ложное срабатывание (выдуманное второе прочтение там, где форма однозначна).
- Перед завершением правки плагина/манифестов: `claude plugin validate .`.

## Чего не делать

- Не утверждать реальные статьи, механику сроков или судебную практику как факт — только флаг.
- Не закреплять одно прочтение молча (в примерах и в `rewrite`); при неясности — два варианта.
- Не разбирать валидность вывода (`pravo-logika`) и убедительность подачи (`pravo-ritorika`) глубже
  флага; не размывать границу «синтаксис / понятия».
- Не давать замечаний без показанного правового результата; вкусовое и типографику, не создающую
  двузначности, — только в блок низкого приоритета «не влияет на смысл».
- Не добавлять реальные дела, клиентов, персональные данные; не использовать англо-американские
  правовые институты как термины (инвариант 5а).
- Не менять лицензию (MIT) без решения владельца; не ломать `LICENSE` и `THIRD_PARTY_NOTICES.md`;
  держать `license: "MIT"` синхронно в трёх манифестах; не убирать ссылку на канал рядом с именем
  автора; не класть `commands/`, `skills/`, `agents/` внутрь `.claude-plugin/`.
