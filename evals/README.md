# E2E-тесты pravo-grammatika

Набор сквозных (end-to-end) тестов: скилл загружается в системный контекст модели, каждому примеру
подаётся вход, а ответ модели сверяется с ожиданием. Проверяет то, ради чего скилл существует, —
**находит ли он реальный дефект формы и не выдумывает ли второе прочтение** там, где формулировка
однозначна.

Набор построен по образцу парных `pravo-logika` / `pravo-ritorika`: 121 условный пример
(87 defect / 34 clean) на все четыре режима и разделы каталога дефектов формы, включая
сделочные сценарии M&A (раздел `K`) и многодефектные тексты (раздел `Y` — два-три
дефекта формы в одном фрагменте). Офлайн-валидатор
структуры и покрытия проходит (`pytest`); реальный прогон через модель — `run_evals.py` на ключе.

## Что внутри

| Файл | Назначение |
|---|---|
| `cases.jsonl` | размеченные примеры (генерируется из `build_cases.py`) |
| `build_cases.py` | источник истины: примеры как Python-данные → пишет `cases.jsonl` |
| `validate_cases.py` | офлайн-проверка структуры и покрытия (без сети, годится для CI/pytest) |
| `run_evals.py` | реальный прогон через Claude API + грейдинг |
| `results.json` | отчёт последнего прогона (создаётся `run_evals.py`) |

## Устройство примера

```json
{"id":"A1","mode":"ambiguity","category":"A.последний_антецедент","difficulty":"easy",
 "input":"Проверь формулировку: «Исполнитель оказывает услуги по ремонту оборудования и программного обеспечения».",
 "expect":"defect","defect_ru":"Проблема последнего антецедента","defect_lat":"",
 "aliases":["последний антецедент","оговорка к последнему или ко всему ряду","объём обязанности размыт"],
 "why":"«По ремонту» относится к последнему члену или ко всему ряду — обязан ли исполнитель ремонтировать ПО."}
```

- **mode** — `ambiguity` / `punctuation` / `norm` / `rewrite`.
- **expect** — `defect` (должен назвать дефект формы из `aliases`) или `clean` (не должен выдумать второе прочтение).
- Все примеры **условны**: реальные нормы и дела не воспроизводятся (инвариант скилла). Скилл проверяет
  **форму** (однозначность), а не валидность вывода (`pravo-logika`), не убедительность (`pravo-ritorika`)
  и не право по существу.

## Покрытие

- **Режимы:** ambiguity, punctuation, norm, rewrite (покрыты все четыре).
- **Разделы** (по первой букве id) — каталог дефектов формы из `references/grammar-errors.md`:
  - `A` — синтаксическая неоднозначность: последний антецедент, присоединение оборота и предложной
    группы, родительная цепочка, эллипсис, косящее обстоятельство, модификатор над рядом;
  - `P` — пунктуация, меняющая смысл: двузначное обособление, статус перечня, тире-условие, скобки,
    режим списка, охват завершающего оборота, кавычки-термин;
  - `S` — область действия: «и/или», перечень без статуса, «в том числе»/«включая, но не
    ограничиваясь», отрицание над рядом, изъятие, эюсдем, границы сроков;
  - `R` — отсылки и анафора: двоящееся местоимение, пустой «соответствующий», съехавшая
    самоотсылка, битая и круговая ссылка, рассогласованный термин, указательное на дистанции;
  - `M` — модальность: «вправе/обязан», «должен», «может», смешение отрицаний, носитель обязанности,
    норма против описания;
  - `O` — нормы языка: висящий оборот, расхождение цифры и прописи, неполная ставка/срок,
    двузначный диапазон, управление, согласование;
  - `W` — переписывание: закрепление заданного прочтения против двух вариантов под выбор автора;
  - `K` — сделки и корпоративные сценарии (право РФ): заверения об обстоятельствах и возмещение
    имущественных потерь, опционы, эскроу, MAC-оговорки, преимущественное право, условия закрытия,
    сделки с заинтересованностью, survival — дефекты формы на материале практики (терминология
    российская, без англо-американских правовых институтов);
  - `Y` — многодефектные тексты: два-три дефекта формы в одном фрагменте;
  - `N` — корректные примеры (форма однозначна): ловят выдуманное второе прочтение.
- **`clean`-примеры** — повтор управляющего слова, «либо…либо», исчерпывающий перечень, явное
  условие вместо обособления, разведённые операторы, заданная граница, разрешимые отсылки, «не обязан»
  без смешения: ловят галлюцинации дефектов формы. Отдельный клин — пограничные примеры (`K5`, `K6`),
  где уместен флаг на соседний скилл, но дефекта **формы** нет.

## Быстрый старт

Офлайн-проверка набора (без ключа и сети). На Windows-консоли с кодировкой cp1251 — с
`PYTHONIOENCODING=utf-8`:

```bash
python evals/validate_cases.py      # отчёт по структуре и покрытию
pytest evals/validate_cases.py      # то же как тест
```

Пересобрать `cases.jsonl` после правки `build_cases.py`:

```bash
python evals/build_cases.py
```

Реальный прогон (нужен ключ). Экспортируйте `ANTHROPIC_API_KEY` или войдите `ant auth login`:

```bash
python evals/run_evals.py --dry-run           # проверить сборку промптов без сети
python evals/run_evals.py --limit 5           # быстрый прогон 5 примеров
python evals/run_evals.py                      # весь набор, грейдер alias, модель claude-opus-4-8
python evals/run_evals.py --grader llm         # грейдинг вызовом-судьёй (точнее для clean)
python evals/run_evals.py --mode ambiguity     # только один режим
```

## Как считается «прошёл»

- **`--grader alias` (по умолчанию, быстрый и бесплатный):** для `defect` — ответ модели содержит
  любой из `aliases` / имя дефекта; для `clean` — есть вердикт «формулировка однозначна / второго
  прочтения нет / дефекта формы нет / переписывать нечего».
- **`--grader llm` (точнее):** отдельный вызов-судья сверяет ответ с ожиданием по существу.
  Рекомендуется для `clean`-примеров: имя дефекта может звучать по-разному, а факт «второе прочтение
  не выдумано» надёжнее оценивает судья, чем поиск подстрок.

## Стоимость и кэширование

Системный контекст (весь скилл) **одинаков для всех запросов** и кэшируется prompt caching'ом —
платится по сути один раз, дальше по цене чтения кэша. Модель по умолчанию — `claude-opus-4-8`,
adaptive thinking + `effort=high` (при старом SDK параметры автоматически опускаются).

## Замечание

Это оценка ФОРМЫ разбора: назван ли нужный дефект формы / не выдумано ли второе прочтение. Юридическую
верность самих формулировок (нормы, практику) скилл не устанавливает и оценка не проверяет — такие
места скилл помечает флагом `[требует проверки]`; понятийные дефекты передаёт `pravo-logika`
(`[→ pravo-logika]`), вопросы подачи — `pravo-ritorika` (`[→ pravo-ritorika]`).
